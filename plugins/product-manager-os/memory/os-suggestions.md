# OS Suggestions

> The self-improvement queue. When the tune-up spots work you repeat (see `activity-log.md`), it writes a proposal here. You **accept / tweak / reject** each one. Nothing is ever auto-applied. Local-only.
>
> Read by the session-start nudge and the weekly tune-up. Resolved items move to the bottom so you keep a record of what the OS has learned.

**Entry format:**
```
### [PENDING] <short title>
- **Kind:** new-skill | tune-skill | fix-memory
- **Signal:** what in the activity log triggered this (e.g. "3× hand-rolled launch emails since Jun 5")
- **Proposal:** the exact change to make
- **Overlap check:** nearest existing skill + why this isn't a duplicate (required for new-skill)
- **Status:** PENDING — awaiting your accept / tweak / reject
```
